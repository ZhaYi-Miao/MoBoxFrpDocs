## 节点怎么选择？

| 考虑内容 | 权重 | 影响内容 | Tip |
| :----: | :----: | :----: | :----: |
| 节点负载 | 高 | 整体流畅度与稳定性 | 超过80%的就可以不考虑了 |
| 节点线路 | 高 | 不同线路连接流畅度与稳定性 |  |
| 是否为海外节点 | 高 | 可用性、流畅性与稳定性 | 冷知识：MossFrp现在没有海外节点 |
| 你对节点的Ping值 | 中 | 基础延迟大小 | 越小越好 |
| 节点与你的距离 | 中 | 整体延迟与稳定性 | / |
| 你的业务内容 | 中 | 可用性 | 部分节点不支持UDP协议（可向节点提供者友好询问） |
| 节点与大多数玩家的距离 | 低 | 部分延迟 | / |

在网页中，节点列表中节点信息可能会带有头衔，以下为各种头衔含义：

  - `【官方节点】`：此类节点为官方提供节点，一般情况下有较高的稳定性与可用性
  - `【优质节点】`：此类节点相对其它节点来说更稳定，质量更高
  
~其实还有个简单粗暴的方法，单位价格越高的节点一般都越好用~
